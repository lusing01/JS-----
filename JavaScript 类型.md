# JS 中的类型 #
## ECMAScript 存在7种简单的基本类型 ##
- ECMAScript 数据类型具有动态性，因此无需再定义其他类型，ECMAScript 不支持任何创建自定义类型的机制。 -
- JS 每一个值都属于某一种数据类型。 -
- 从变量、参数、返回值到表达式中间结果，任何在 JS 运行过程中产生的数据都具有：运行时类型(也即7类)。 -
- 在 JS 运行时的类型就是代码实际执行过程中用到的类型(即7类)。 -
- 类型广泛使用于变量、函数参数、表达式、函数返回值等场合。 -
- 新的ECMAScript标准定义了7种语言类型，加入了 Symbol 类型。 -
## Undefined 类型 ##
- Undefined 类型表示未定义，只有一个特殊的 undefined 值。 -
- undefined 值在用 var 关键字声明未初始化的变量时，该变量的值即为 undefined。 -
- 一般来说，undefined 的值不需要进行显式的设置，其存在主要目的是用于比较：为了区分空对象指针与未初始化的变量。 -
    var message; // 这个变量声明之后默认取得了 undefined 值
    // 下面这个变量并没有声明
    // var age
    alert(message); // "undefined"
    alert(age); // 产生错误
- 对于尚未声明的变量，只能用 typeof 检测其数据类型。 -
### 关于使用 viod 0 来代替 undefined ###
- 由于 JS 语言的失误之一，JS 代码中的 undefined 并非是关键字，而是一个变量。 -
- 为了避免被篡改，可使用 viod 0 来获取 undefined  值。 -
## Null 类型 ##
- Null 类型同样只有一个特殊的值 null 的类型。 -
- null 表示一个空指针，使用 typedef 检测 null 值会返回 object。 -
- null 是 JS 的关键字，表示“定义了但为空”，语义表示空值，可直接使用关键字获取 null 值。 -
- 如果定义的变量准备在将来用于保存对象，最好将该变量初始化为 null 而不是其他值(检查null值就可知道该变量是否保存了一个对象的引用)。 -
- 与 undefined 值不同，null 在使用时，需要显式的进行设置。 -
## Boolean 类型 ##
- Boolean 类型是 ECMAScript 中使用得最多的一种类型，该类型只有 true 和 false 两种字面值。 -
- Boolean 类型的值不能等同于数字值。 -
- Boolean 类型的值区分大小写。 -
- 可以对任何数据类型的值调用 Boolean() 函数，而且总会返回一个 Boolean 值，返回的值，取决于要转换值的数据类型及其实际值。 -
## String 类型 ##
- String 类型用于表示由零或多个 16 位 Unicode 字符组成的字符序列，即字符串的 UTF16 编码。 -
- String 类型用于表示文本数据，有最大长度 2^53 - 1，该最大长度并非字符数。 -
- 字符串的最大长度，受表示其的编码长度影响。 -
- JS 字串符将每个 UTF16 单元当做一个字符来处理，所以处理超过基本字符区域的应当注意。 -
- ECMAScript 中的字符串是不可变的，也就是说，字符串一旦创建，它们的值就不能改变。 -
### 转换为字符串 ###
- 其一，可以使用除 null 值和 undefined 值外其他值都有的 toString() 方法。 -
	- 通过指定基数， toString() 方法会改变输出的值。 -
	- 在不知道要转换的值是不是 null 或 undefined 的情况下，还可以使用转型函数 String() ，这个函数能够将任何类型的值转换为字符串。 -
- 其二，可以使用加法运算符：使用加号操作符把待转换值与一个字符串 "" 加在一起。 -
## Number 类型 ##
- Number 类型使用 IEEE754 格式来表示整数和浮点数值。 -
- JS 中有 +0 和 -0 之分，运算时注意在除法中的应用。 -
- 浮点数值：该数值中必须包含一个小数点，并且小数点后面必须至少有一位数字。 -
	- 保存浮点数值需要的内存空间是保存整数值的两倍，因此 ECMAScript会不失时机地将浮点数值转换为整数值。 -
	- 科学计数法在 JS 中依然适用。 -
- 由于内存的限制，JS 中的数值存在范围：2^64 - 2^53 + 3 个值。 -
- 如果某次计算结果得到一个超出数值范围的值，该数值会被自动转换成特殊的 Infinity 值，负数：-Infinity，正数： Infinity。 -
- Infinity 值无法继续参与下一次计算。 -
- NaN：not a number，一个特殊的数值，用于表示一个本来要返回数值的操作数未返回数值的情况。 -
	- 任何数值除于0会返回 NaN。 -
	- 任何涉及 NaN 的操作都会返回 NaN。 -
	- NaN 与任何值都不相等，包括自身。 -
- 非整数的 Number 类型无法用 == 来进行比较：0.1 + 0.2 == 0.3 会输出false。 -
- 比较非整数的正确方式：使用 JS 提供的最小精度值：Number.EPSILON。 -
## Symbol 类型 ##
- Symbol 类型是 ES6 中引入的新类型，它是一切非字符串的对象 key 的集合。 -
- 整个对象系统被用 Symbol 重塑。 -
- Symbol 可具有字符串类型的描述，但即使描述相同，Symbol也不相等。 -
- 创建 Symbol 使用全局的 Symbol 函数： Symbol()。 -
- 我们可以使用 Symbol.iterator 来自定义 for...of 在对象上的行为。 -
- 直接用 new 调用 symbol 函数会抛出错误。 - 
## Objcet 类型 ##
- Objcet 其实就是一组数据和功能的集合(属性的集合)。 -
- 在 ECMAScript 中，Object 类型是所有它的实例的基础。 -
- Object 类型所具有的任何属性和方法也同样存在于更具体的对象中，甚至在原型上添加方法，都可以用于基本类型。 -
- JS 中的“类”仅仅是运行时对象的一个私有属性。 -
### 为什么给对象添加的方法能用在基本类型上 ###
- `.运算符`提供了装箱操作，会根据基础类型构造一个临时对象，使得我们能在基础类型上调用对应对象的方法。 -
### 对象的属性和方法 ###
- constructor：保存着用于创建当前对象的函数。 -
- hasOwnProperty(propertyName)：检查给定的属性是否存在于当前对象实例中。 -
- isPrototypeOf(object)：用于检查传入的对象是否是传入对象的原型。 -
- propertyIsEnumerable(propertyName)：用于检查给定的属性是否能够使用 for-in 语句来枚举。 -
- toLocaleString()：返回对象的字符串表示，该字符串与执行环境的地区对应。 -
- toString()：返回对象的字符串表示。 -
- valueOf()：返回对象的字符串、数值或布尔值表示。 -
## 类型转换 ## 
- JS 是弱类型语言，类型转换发生的非常频繁。 -
- 建议进行显式的类型转换后，用 === 来比较。 -
### StringToNumber ###
- 3 个函数可以把非数值转换为数值： Number()、 parseInt() 和 parseFloat()。 -
- 转型函数 Number() 可以用于任何数据类型，而另两个函数则专门用于把字符串转换成数值。 -
- 转型函数的转换规则： -
	- Boolean 值，true：1，false：0 。 -
	- 数字值，只是简单的传入和返回。 -
	- null：0，undefined：NaN。 -
	- 字符串：数字(包含十六进制将转换为十进制、为空则转换为0)、浮点数值、NaN(不包含数字和浮点数)。 -
	- 对象：调用对象的 valueOf() 方法。 -
		- 若转换的结果为 NaN，则调用对象的 toString() 方法。 -
- 处理整数的时候更常用的是 parseInt() 函数。 -
	- 会忽略字符前的空格。 -
	- 如果第一个非空格字符不是数字或负号，返回 NaN。 -
	- 如果第一个是数字，继续解析第二个，直到解析完或遇到第一个非数字符。 -
	- 字符串中的第一个字符是数字字符，parseInt() 能够识别出各种整数格式。 -
	- 可提供第二个参数(即转换时使用的基数)，来消除转换中数制的迷惑。 -
- parseFloat() 函数 -
	- parseFloat() 也是从第一个字符(位置 0)开始解析每个字符，一直解析到字符串末尾或无效的浮点数数字。 -
	- 字符串中的第一个小数点是有效的，而第二个小数点就是无效的。 -
	- 始终都忽略前导的零，只解析十进制值。 -
	- 如果字符串包含的是一个可解析为整数的数(没有小数点，或者小数点后都是零)，parseFloat() 会返回整数。 -
### NumberToString ###
- 较小范围里，数字到字符串的转换是符合直觉的十进制表示。 -
- 当 Number 绝对值较大或较小时，使用科学计数法表示。 -
### 装箱转换 ###
- 将基本类型转换为对应的对象，类型转换中一种相当重要的种类。 -
- 装箱机制会频繁的产生临时对象。 -
- 使用内置的 Object 函数(constructor)，可以 JS 代码中显示调用装箱能力。 -
- 每一类装箱对象皆有私有的 Class 属性，可用 Object.prototype.toString 来获取。 -
- Object.prototype.toString 是准确识别对象对应基本类型的方法。 -
- 函数的 call 方法本身会产生装箱操作，需配合 typeof 来区分基本类型还是对象类型。 -
### 拆箱转换 ###
- JS 标准中，规定 ToPrimitive 函数：对象类型到基本类型的转换。 -
- 对象到 String 和 Number 的转换都遵循“先拆箱再转换”的规则。 -
- 拆箱转换会尝试调用 valueOf 和 toString 来获取拆箱后的基本类型。 -
	- valueOf 和 toString 都不存在或没有返回基本类型，产生 TypeError。 -
- 在 ES6 之后，还允许对象通过显式指定 @@toPrimitive Symbol 来覆盖原有的行为。 -
